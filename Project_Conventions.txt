NAMING:
BEM + styling:
Context = A page-specific single use container; always combined with a block; used for naming modifiers within the block
e.g. --workflow

Blocks = multi-use, inner geometry (e.g. flex-direction, padding, gaps, ...)
e.g. default-panel

Elements = block-dependent, outer geometry (e.g. margins)
e.g. default-panel__info-container

Modifiers = manipulates blocks or elements geometry
Modifier hierarchy:
1. general modifiers; e.g. .header--dark
2. context dependent modifiers; e.g. .flex-tags__tag--[context-name]
3. state modifiers; e.g. .button--[state-name]

MEDIA-Placement:
wrapper around each media for placement with corresponding placement class e.g. .media--rounded, .media--contain
    reason: img-elements themselves change their sizing depending on their placement, thus they can't have backgrounds
